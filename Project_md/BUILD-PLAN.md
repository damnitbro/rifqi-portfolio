# Portfolio Build Plan

## Phase 1 — Positioning

- Choose one primary title
- Define the roles you are applying for
- Select four projects
- Remove weak, repetitive, or unfinished work

## Phase 2 — Content

- Write each case study
- Verify responsibilities and credits
- Verify all numbers
- Prepare thumbnails and hero images
- Export process visuals
- Prepare the latest resume

## Phase 3 — Design

- Create mobile wireframes first
- Define typography
- Define spacing and grid
- Design project cards
- Design one complete case-study template
- Use restrained motion

## Phase 4 — Development

Recommended stack:

- Next.js
- TypeScript
- Tailwind CSS
- Framer Motion
- Vercel
- Markdown or MDX for project content

Suggested structure:

```text
app/
  page.tsx
  work/
    page.tsx
    [slug]/
      page.tsx
  about/
    page.tsx
  contact/
    page.tsx

content/
  HOME.md
  ABOUT.md
  projects/
    kemenkes.md
    soca.md
    real-estate.md
    personal-3d.md

public/
  images/
  videos/
  resume/
```

## Phase 5 — Quality Control

- Test on mobile, tablet, and desktop
- Check every link
- Compress images and video
- Add alt text
- Add metadata and Open Graph images
- Test keyboard navigation
- Support reduced motion
- Run Lighthouse
- Connect a custom domain

## Core Rule

The website should communicate:

> This is what I am best at, this is how I think, and this is the value my work creates.
